0.1. Un poco de historia

0.1.1. 1978 - Donald Knuth publica TeX

  • TeX como un sistema tipográfico de bajo nivel para producir documentos elegantes e independientes de la impresora (serie: The Art of Computer Programming
  • TeX funciona con "primitivas" (cajas, pegamento, penalizaciones), potentes pero engorrosos.

0.1.2. 1984 - Leslie Lamport publica LaTeX 2.09

  • Creo capa de alto nivel alrededor de TeX (i.e. LaTeX)
  • Clases de Documentos (article, report, etc.)
  • Ambientes:
    • Listas
    • Tablas (Cuadros)
    • Figuras
    • Secciones

0.1.3. 1989 - AMS/LaTeX

  • Extensiones para escribir matemáticas
    • amsmath
    • amsthm
    • amssymb

0.1.4. 1994 Publicación de LaTeX2e

  • Revisión compatible desarrollada por el Proyecto LaTeX3 (Frank Mittelbach, Chris Rowley, Rainer Schöpf, et al.).
  • Sistema modular de clases y paquetes que sustituyó al código monolítico de LaTeX 2.09.
  • Comandos \ProvidesClass, \ProvidesPackage y mecanismo robusto de gestión de paquetes
  • LPPL (LaTeX Project Public License) como licencia oficial del proyecto.
  • Versión actual instalada por defecto en TeX Live, MikTeX, MacTeX, etc.

0.1.5. 2000 - 2020's Adopción masiva y ecosistema

  • CTAN: repositorio centralizado de miles de paquetes TeX / LaTeX
  • Democratización del acceso mediante editores colaborativos en línea (e.g. Overleaf, ShareLaTeX)
  • Integración con flujos de trabajo modernos:
    • control de versiones (Git)
    • Integración Continua (latexmk)
    • Investigación Reproducible (Sweave, knitr, Jupyter)
  • Uso más allá de matemáticas y física: por ejemplo: biología, economía, lingüística, derecho y editoriales comerciales

1. ¿ Por qué LaTeX ?

1.1. Separación de Contenido y Presentación

Los autores escriben el contenido; LaTeX decide la presentación (similar a HTML/CSS, pero para PDF de calidad tipográfica).

1.2. Tipografía

Los algoritmos originales de Knuth para división de líneas, separación silábica y espaciado matemático siguen siendo insuperables en documentos técnicos.

1.3. Estabilidad y Reproducibilidad

Un documento de LaTeX compilado hoy produce una salida prácticamente idéntica a la de hace una década (salvo actualizaciones de paquetes), esencial para la conservación de la investigación.

1.4. Comunidad y Ecosistema

  • Miles de paquetes gratuitos
  • documentación exhaustiva (LaTeX Companion, TeXbook)
  • Soporte activo vía listas de correo, StackExchange y CTAN.
  • Código Abierto y Licenciamiento
    • Distribuciones
      • TexLive (Linux)
      • MiKTeX (Windows)
      • MacTex (MacOS)
