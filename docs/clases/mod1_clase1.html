1. Anatomía genérica de una computadora

  • Unidad Central de Procesamiento (CPU): El “cerebro” que ejecuta cálculos y controla operaciones.
  • Memoria de Acceso Aleatorio (RAM): Almacenamiento temporal de alta velocidad para datos en uso activo.
  • Almacenamiento (HDD/SSD): Espacio persistente para guardar programas y archivos a largo plazo.

  • Dispositivo semiconductor que actúa como un interruptor (switch) o amplificador en circuitos electrónicos.

2.4. Arquitectura von Neumann: Modelo Fundamental de Cómputo

  • ALU (Unidad Aritmético-Lógica): Ejecuta operaciones matemáticas y lógicas básicas (sumas, comparaciones, etc.).
  • Unidad de Control: Coordina el flujo de datos e instrucciones, decodificando comandos desde la memoria.
  • Memoria Principal: Almacena datos e instrucciones del programa en una única unidad.
  • Dispositivos de Entrada (Input): Introducen información al sistema (ej.: teclado, sensor, archivo).
  • Dispositivos de Salida (Output): Presentan resultados procesados (ej.: pantalla, altavoz, archivo).

2.5. Arquitectura de Von Neuman

Tomado de C. Jacob, 2000

  • Cómputo universal: Diseño independiente de problemas específicos mediante colecciones de componentes lógicos (AND, OR).
  • Algoritmo y datos residen en la misma memoria (i.e. “programa almacenado”).
  • Cada ubicación de memoria tiene una dirección única para acceso directo.
  • Los comandos del programa se almacenan secuencialmente, con posibilidad de saltos condicionales.

2.7. Ensamblador

  • Código máquina: secuencia binaria ejecutada directamente por CPU.
  • Específico por arquitectura: cada CPU (x86, ARM, incluso dif. entre modelos) interpreta códigos distintos (incompatibilidad entre plataformas).
  • casi imposibles de depurar manualmente, uso de debuggers hexadecimales (ingeniería inversa).

3. Lenguajes y su Relación con el Código Máquina

Existen dos tipos de lenguajes de programación:

3.1. Lenguajes Compilados

Utilizan un compilador para convertir todo el código a lenguaje de máquina.

  • Un compilador es un programa que traduce código fuente lenguaje de alto nivel a código máquina ejecutable por el CPU. Ejemplos:
    • C
    • Rust
    • Julia
  • Realiza la traducción completa antes de la ejecución, generando un archivo objeto independiente (ejecutable binario).
  • El código máquina resultante es específico de la arquitectura del procesador (ej.: x86, ARM).
  • Optimiza instrucciones para mejorar rendimiento, atado al hardware donde se compiló.
  • Elimina la necesidad de escribir en binario/hexadecimal, facilitando el desarrollo.
  • Explica por qué el mismo código fuente requiere recompilación para distintas plataformas.

3.2. Lenguaje Interpretado

  • Lenguaje de programación que no se compila a código máquina antes de ser ejecutado, sino que se interpreta línea por línea en tiempo de ejecución.
  • Presenta mayor flexibilidad, permitiendo cambios rápidos en el código sin necesidad de recompilación.
  • Ejemplos incluyen:
    • Python
    • R
    • JavaScript,
  • Permite depuración en tiempo real, facilitando la identificación de errores durante la ejecución.
  • Suele ser más lento que los lenguajes compilados debido al proceso de interpretación.
  • Utiliza un intérprete que traduce y ejecuta el código sobre la marcha, en contraste con la generación previa de código máquina.

4. ¿ Qué es un sistema Operativo?

  • Software base que gestiona el hardware, recursos del sistema y actúa como intermediario entre aplicaciones y componentes físicos.
  • Proporciona servicios esenciales: administración de procesos, memoria, almacenamiento y dispositivos periféricos (red, teclado, sonido, pantalla, etc).
  • Garantiza la ejecución segura y concurrente de múltiples programas mediante mecanismos de control de acceso.

5. Proyecto GNU

5.1. Aplicaciones

  • En 1983, Richard Stallman anunció en Usenet la creación del proyecto GNU, con el objetivo de desarrollar un sistema operativo completamente libre, como alternativa al software propietario.
  • En 1989, Stallman publicó la Licencia Pública General de GNU (GPL), una licencia que garantiza estas libertades y evita el uso no ético del software.

from wikimedia.org “RichardStallmanatLibrePlanet2019” by Ruben Rodriguez

5.2. Las cuatro libertades de Software Libre

  • Libertad para usar el software
  • Libertad para estudiar el código fuente,
  • Libertad para adaptarlo a nuestras necesidades
  • Libertad para compartir las modificaciones con la comunidad

A beautiful Gnu from: https://gnu.org

  • La GPL se ha convertido en una de las licencias de software libre más utilizadas, fomentando el desarrollo colaborativo y la cultura del “copyleft”.
  • El proyecto GNU incluye herramientas fundamentales como el compilador GCC y el editor Emacs, que se han vuelto esenciales en el mundo del software libre.
  • Para más detalles, puedes leer el anuncio original aquí: [Anuncio del Proyecto GNU](https://www.gnu.org/gnu/initial-announcement.html).

5.3. El Kernel Linux

5.3.1. Anuncio de Linus Torvalds y el Nacimiento de Linux

  • En 1991, Linus Torvalds publicó un mensaje en Usenet anunciando el desarrollo de un núcleo operativo (kernel) llamado Linux, inspirado en los sistemas Unix y en soporte para el software GNU.
  • Inicialmente, Linux fue creado como un proyecto personal, pero rápidamente ganó popularidad al ser un sistema operativo libre que podía usarse junto con las herramientas GNU.

image of Linus Torvalds speaking at the LinuxCon Europe 2014 in Düsseldorf

  • La combinación de GNU con el núcleo Linux dio origen a “GNU/Linux”, un sistema operativo robusto, libre y de código abierto, ampliamente utilizado en servidores y dispositivos.
  • Este sistema operativo se convirtió en la base de muchas distribuciones populares (ej.: Ubuntu, Debian, CentOS), que son el estándar en servidores y nube, corriendo en más del 90% de los servidores globales.
  • La filosofía de software libre promovida tanto por Stallman como por Torvalds ha impulsado un ecosistema colaborativo que sigue moldeando la informática actual.

6. GNU/Linux

6.1. Comandos básicos I

  • ls: Listar contenido de directorios (ej.: ls -l para detalles en formato largo, -a para archivos ocultos).
  • cd: Navegar entre carpetas (ej.: cd ../ para subir un nivel o cd ~ para ir al directorio personal).
  • pwd: Mostrar ruta absoluta del directorio actual (útil al trabajar con rutas relativas o absolutas).
  • mkdir: Crear nuevos directorios (ej.: mkdir AnalisisDatos para organizar proyectos).
  • cp/mv: Copiar (cp archivoorigen destino) o mover/renombrar (mv antiguo.csv nuevo.csv) archivos.
  • rm: Eliminar archivos (ej.: rm temporal.txt; usar -r para carpetas con extrema precaución).

6.2. Ejercicios

  • Buscar ayuda con man y --help
  • Usar Pipes (|) y redireccionamientos (><) para crear nuevos archivos
